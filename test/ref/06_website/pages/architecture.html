架构设计

四层架构设计，从底层通信到顶层应用，层次清晰、职责分明

架构总览

Starletto 采用分层架构设计，每一层都有明确的职责与接口定义

📱

应用层 (Application Layer)

感知、规划、控制等核心算法模块，提供完整的自动驾驶功能实现

⚙️

运行时层 (Runtime Layer)

进程管理、调度器、触发器等运行时基础设施，保障系统稳定运行

🚀

传输层 (Transport Layer)

高性能通信框架，支持单进程零拷贝与跨进程共享内存传输

📊

模式层 (Schema Layer)

数据模式定义与代码生成，确保数据一致性与类型安全

模式层 (Schema Layer)

数据模式定义与代码生成，构建类型安全的数据基础

📝

Schema 定义

使用 .schema 文件定义数据结构。 支持基本类型、复合类型、枚举等。 清晰的语法规则与默认值定义。

⚙️

代码生成

自动生成 C++ 数据结构代码。 支持序列化与反序列化。 类型安全的数据访问接口。

📦

值文件

运行时配置数据存储。 支持热更新与版本管理。 与 Schema 严格对应。

🔒

类型安全

编译期类型检查。 避免运行时类型错误。 提供完整的类型推导。 

// 示例：感知目标数据结构定义
message PerceptionObject {
    uint32 id = 1;
    string type = 2;
    double position_x = 3;
    double position_y = 4;
    double position_z = 5;
    double velocity_x = 6;
    double velocity_y = 7;
    double heading = 8;
}

传输层 (Transport Layer)

高性能通信框架，支持多种传输模式

⚡

单进程通信

进程内零拷贝数据传输。 指针传递，无序列化开销。 极低延迟，微秒级响应。

🔗

跨进程通信

共享内存零拷贝传输。 Loan/View 机制避免数据拷贝。 高效的进程间数据共享。

📡

Channel 抽象

统一的 Channel 接口。 支持 Reader/Writer 模式。 自动选择最优传输方式。

🔄

Record & Replay

数据记录与回放功能。 支持多通道同步回放。 便于调试与测试。 

// 示例：Channel 使用
auto channel = Transport::CreateChannel<PerceptionObject>("/perception/objects");

// Writer 发送数据
auto writer = channel->CreateWriter();
writer->Write(object);

// Reader 接收数据
auto reader = channel->CreateReader();
reader->Observe([](const PerceptionObject& obj) {
    // 处理感知数据
});
